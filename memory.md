## Challenges Faced & How We Solved Them

1. **Lottie-React Blank Page Crash (Import Interop Error)**
   - **The Problem**: Upon adding the Lottie animation, the app crashed to a blank screen. The console indicated `Element type is invalid: expected a string... but got: object`.
   - **The Fix**: This happened due to how Vite (ESM) interacts with CommonJS defaults in `lottie-react`. We solved it using an interop fallback: `const Lottie = LottieLib.default || LottieLib;`.

2. **React-Markdown Dependency Conflicts**
   - **The Problem**: Attempting to `npm install react-markdown` resulted in `ERESOLVE` errors because of conflicting `vite` and `react` peer dependencies in this specific workspace setup.
   - **The Fix**: Bypassed the strict version check by forcing the installation via `npm install react-markdown --legacy-peer-deps`.

3. **Backend Not Hot-Reloading**
   - **The Problem**: While the Vite frontend hot-reloads instantly, the Express backend was running via a simple `node index.js` command, meaning new API routes (like `/api/admin/apply-ai-changes`) were returning 404s.
   - **The Fix**: Used terminal commands (`netstat -ano | findstr :5000` and `taskkill`) to manually hunt down and kill the lingering node processes, then rebooted the backend.

4. **ClassTimetablePage Loading State Miss**
   - **The Problem**: When replacing the `Loader2` spinners globally, `ClassTimetablePage` was missed because it was using a raw CSS spinner (`border-spin`) rather than the Lucide icon.
   - **The Fix**: Used `findstr` to search for "loading" text globally, found the exact file, and swapped the CSS spinner out for the `Lottie` animation.
